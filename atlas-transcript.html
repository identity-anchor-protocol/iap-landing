This page documents a real beta session: an AI agent worked through the IAP flow, encountered a key-custody continuity mismatch, and completed offline verification.

This transcript is included as a narrative example. The authoritative product claims remain the current beta docs and protocol notes.

Why it matters

The session makes the core beta idea concrete: identity continuity is no longer assumed, it can be checked.

The value of the transcript is not that it proves more than the protocol does. It shows a real operator lesson from beta: a reused key means a reused agent_id, and that affects continuity.

What the transcript demonstrates accurately

  • IAP beta binds identity to an Ed25519 keypair.
  • Reusing a key means reusing the same agent_id.
  • Continuity requires a strictly increasing sequence.
  • Offline verification is a real part of the current beta flow.

What it does not change

  • It does not expand the product claims beyond the current beta scope.
  • It does not imply behavioral attestation.
  • It does not imply deterministic outputs.
